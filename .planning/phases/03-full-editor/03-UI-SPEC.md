# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 03: Full Editor.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase transforms the read-only viewer into a full editor. All decisions below
> are locked from 03-CONTEXT.md (D-01 through D-15) and the existing codebase.
> No new colors or typography scales are introduced — all values reuse Phase 01 tokens.

| Property | Value |

| Tool | none (manual Tailwind v4 token system) |

Primary visual anchor: the selected node's canvas card with its solid accent selection ring; all other elements recede visually to draw attention to the active node.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline button padding |
| sm | 8px | Compact element spacing, metadata row gap, context menu item padding-y |
| md | 16px | Panel section padding, context menu horizontal padding |
| lg | 24px | Panel section breaks |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Context menu item height: 28px (7px padding-y × 2 + content) — nearest 4px boundary for touch ergonomics; checker-accepted ergonomics exception
- Inline rename input: height matches node title line (approximately 20px computed); positioned exactly over node via inverse D3 zoom transform — height driven by content, not spacing scale
- Status bar height: 32px (established in Phase 02 StatusBar.tsx — do not change)
- Panel header: min-h-[52px] (established in Phase 02 SidePanel.tsx — do not change)
- Touch targets for [+] / [x] metadata buttons: minimum 24px × 24px

Source: index.css, SidePanel.tsx, StatusBar.tsx measured values; 03-CONTEXT.md D-01.

## Color

| Role | Token | Dark theme value | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | --rv-bg-canvas / --rv-bg-base | #131313 | Canvas background, app base |
| Secondary (30%) | --rv-bg-panel / --rv-bg-surface | #1b1b1c | Side panel, context menu surface, toolbar, modals, toasts |
| Accent (10%) | --rv-accent | #4a9eff | Selection ring (solid 2px outline), segmented toggle active segment, [E] edit button icon on hover, copy-ID button on hover, focus ring color (dashed variant), context menu keyboard focus highlight |
| Destructive | --rv-status-blocked | #ef4444 | Delete menu item text, confirmation dialog destructive button, save error indicator in status bar, 3rd-failure modal warning icon |

Accent reserved for:
1. Node selection ring — `outline: 2px solid var(--rv-accent)` (already established Phase 02)
2. Node focus ring — `outline: 2px dashed var(--rv-accent)` (NEW, coexists with selection ring per D-06)
3. Context menu keyboard-focused item background — `var(--rv-accent-muted)` background + `var(--rv-accent)` text
4. Segmented toggle active segment background — `var(--rv-accent-muted)`; active segment text — `var(--rv-accent)`
5. [E] edit button icon on hover state
6. Panel title field focus border — `var(--rv-border-focus)` (same value as accent, #4a9eff)
7. Inline rename input border — `var(--rv-border-focus)` on focus
8. Metadata row input border on focus — `var(--rv-border-focus)`

DO NOT use accent for:
- Context menu item text (use --rv-text-primary)
- Metadata keys or values at rest (use --rv-text-secondary)
- Status bar save indicator "Saved" state (use --rv-status-completed green)
- Modal body text

Additional semantic tokens for this phase:
- Save state "Saved": `--rv-status-completed` (#4ade80) — dot + text
- Save state "Saving": `--rv-text-secondary` (#999999) — animated ellipsis, muted
- Save state "Error": `--rv-status-blocked` (#ef4444) — exclamation icon + red text
- Toast background: `--rv-bg-elevated` (#252527)
- Modal overlay: `rgba(0, 0, 0, 0.6)` (not a token — inline value only)
- Modal surface: `--rv-bg-elevated` (#252527)
- Focus ring (dashed): `outline: 2px dashed var(--rv-accent); outline-offset: 2px` — disappears on mouse click, reappears on first keyboard press via `:focus-visible` behavior

Source: index.css full token definitions; 03-CONTEXT.md D-06, D-14, D-15; RoadmapNode.tsx isSelected pattern.

## Component Inventory

New components introduced this phase. Each must use only --rv-* tokens and inline SVG icons.

### ContextMenu

Source: 03-CONTEXT.md D-01, D-02, D-03, EDIT-09, EDIT-18.

**Structure:** Custom `<div>` rendered in the webview (not native OS menu). Single DOM mount point at document root to escape stacking contexts. Positioned via `position: fixed` with viewport boundary clamp.

**Anatomy:**
```
[ContextMenu root div]
  [MenuGroup]               — visual separator between groups
    [MenuItem]              — single action
    [MenuItem with hint]    — action + keyboard shortcut label
  [Separator hr]
  [MenuGroup]
    [SubMenuTrigger]        — Change Status ▶  (only sub-menu)
      [SubMenu]
        [StatusMenuItem]    — one per statusConfig entry
```

**Node right-click menu — full action set (in order):**
```
Rename                    F2
─────────────────────────────
Add Child                 Enter
Add Sibling Above         Shift+Enter
Add Sibling Below         Tab
─────────────────────────────
Duplicate                 Ctrl+D
Copy                      Ctrl+C
Paste                     Ctrl+V
─────────────────────────────
Move Up                   Ctrl+↑
Move Down                 Ctrl+↓
─────────────────────────────
Change Status             ▶
─────────────────────────────
Delete                    Del        ← red text color (--rv-status-blocked)
```

**Canvas background right-click menu (no node under cursor):**
```
Paste                     Ctrl+V
Add Root Child            Enter
─────────────────────────────
Fit to View
Toggle Layout
```

**Visual spec:**
- Background: `var(--rv-bg-elevated)` (#252527 dark)
- Border: `1px solid var(--rv-border)` (#2a2a2c dark)
- Border-radius: 8px
- Box-shadow: `var(--rv-shadow-config)` (0 4px 20px rgba(0,0,0,0.5))
- Min-width: 200px; max-width: 280px
- Padding: 4px 0 (top/bottom inset for grouped items)
- Render budget: must be visible ≤ 50ms from right-click event (per EDIT-09)

**MenuItem:**
- Height: 28px
- Padding: 0 12px
- Display: flex; align-items: center; justify-content: space-between
- Label: 13px / 400 / `var(--rv-text-primary)`
- Shortcut hint: 11px / 400 / `var(--rv-text-tertiary)`; right-aligned
- Hover state: background `var(--rv-bg-hover)` (#2e2e30 dark), no border change
- Keyboard focus state (arrow key navigation): background `var(--rv-accent-muted)`, text `var(--rv-text-primary)` — NOT the same as hover
- Delete item: label color `var(--rv-status-blocked)` (#ef4444); shortcut hint color `var(--rv-status-blocked)` at 60% opacity
- Disabled item (e.g., Paste when clipboard empty): text `var(--rv-text-tertiary)`, no hover state, cursor: default

**Separator:**
- `<hr>` element: `1px solid var(--rv-border-subtle)`, margin 4px 0

**SubMenuTrigger (Change Status):**
- Same as MenuItem layout; right side shows "▶" chevron icon (12×12 inline SVG) instead of shortcut hint
- Sub-menu opens to the right; if viewport clips right edge, opens to the left
- Sub-menu shares same visual spec as root menu

**ARIA roles:**
- Root: `role="menu"`, `aria-label="Node actions"`
- Canvas root: `role="menu"`, `aria-label="Canvas actions"`
- Each item: `role="menuitem"`
- Sub-menu trigger: `role="menuitem"`, `aria-haspopup="true"`, `aria-expanded="true/false"`
- Sub-menu: `role="menu"`, `aria-label="Change status"`
- Separator: `role="separator"`
- Focus management: first item receives focus on open; arrow keys cycle items; Escape closes and returns focus to triggering element

**Animation:** `opacity: 0 → 1` + `transform: scale(0.97) → scale(1)` in 80ms ease-out. Must complete within the 50ms render budget — mount immediately, animate from already-visible position.

### ConfirmationDialog

Source: 03-CONTEXT.md Claude's Discretion ("Confirmation dialog design for non-leaf delete — must show child count"); EDIT-03.

**Trigger:** Delete action on a node that has children.
**NOT shown for:** Leaf node deletion (immediate, no dialog).

**Structure:**
```
[Modal overlay div — full screen, rgba(0,0,0,0.6)]
  [Dialog surface]
    [Icon — warning triangle, --rv-status-blocked color]
    [Heading] "Delete node and {N} children?"
    [Body] "This will permanently remove "{node title}" and all {N} nodes in its subtree. This cannot be undone."
    [Button row]
      [Keep Node button]  [Delete button — destructive]
```

**Visual spec:**
- Overlay: `position: fixed; inset: 0; background: rgba(0,0,0,0.6); z-index: 9999`
- Surface: `background: var(--rv-bg-elevated)`, `border: 1px solid var(--rv-border)`, `border-radius: 12px`, `box-shadow: var(--rv-shadow-config)`, width 400px, max-width calc(100vw - 48px)
- Surface padding: 24px
- Icon: 20×20 inline SVG triangle-warning, color `var(--rv-status-blocked)`
- Heading: 14px / 600 / `var(--rv-text-primary)`; margin-top 12px
- Body: 13px / 400 / `var(--rv-text-secondary)`; line-height 1.5; margin-top 8px
- Button row: margin-top 20px; `display: flex; justify-content: flex-end; gap: 8px`

**Keep Node button:**
- Background: `var(--rv-bg-hover)`, border: `1px solid var(--rv-border)`, border-radius: 6px
- Padding: 8px 16px; 13px / 400 / `var(--rv-text-primary)`
- Hover: `background: var(--rv-bg-active)`

**Delete button (destructive):**
- Background: `var(--rv-status-blocked)` (#ef4444), border: none, border-radius: 6px
- Padding: 8px 16px; 13px / 600 / `#ffffff`
- Hover: background slightly darker (`rgba(239,68,68,0.85)`)

**ARIA / keyboard:**
- `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading ID

- Initial focus: Keep Node button (safer default — prevents accidental delete on Enter)

### SaveIndicator (StatusBar extension)

Source: 03-CONTEXT.md D-15; EDIT-15.

The existing StatusBar right section gains a save state indicator. Three states:

**Saved state:**
- Dot: `var(--rv-status-completed)` (#4ade80) — 7px circle (matches existing Connected dot)
- Text: "Saved" — 11px / 400 / `var(--rv-text-tertiary)`

**Saving state:**
- Animated spinner or pulsing dot: `var(--rv-text-secondary)` (#999999)
- Text: "Saving…" — 11px / 400 / `var(--rv-text-secondary)`
- Animation: 200ms fade between dots or CSS spinner at 16fps; respect `prefers-reduced-motion`

- Icon: `!` exclamation mark (inline SVG, 10×10), color `var(--rv-status-blocked)`

**Error state — 2nd failure (manual retry):**
- Same icon + color
- Text: "Error saving — click to retry" — 11px / 400 / `var(--rv-status-blocked)`
- Entire indicator is a `<button>` with cursor: pointer; hover underline on text
- `aria-label="Save failed. Click to retry saving."`

**Error state — 3rd failure:**
- Opens SaveFailureModal (see below); status bar shows "Error saving" persistently

**Placement:** Right section of status bar, leftmost item in that section (before node count).

### FocusRing (dashed variant — canvas navigation)

Source: 03-CONTEXT.md D-06.

The existing selection ring on `RoadmapNodeCard` uses solid `outline: 2px solid var(--rv-accent)`.

The new focus ring for keyboard navigation uses:
```css
outline: 2px dashed var(--rv-accent);
outline-offset: 2px;
```

**Coexistence rules:**
- Node is **focused only** (arrow-key navigation, not yet selected): dashed accent outline
- Node is **selected only** (Space pressed, panel open, no keyboard focus): solid accent outline
- Node is **both focused and selected**: both rings applied — dashed ring at `outline-offset: 4px`, solid ring at `outline-offset: 1px`. Implemented via `box-shadow` for the second ring to avoid CSS outline stacking limitation.
  - Exact: `outline: 2px solid var(--rv-accent); outline-offset: 1px; box-shadow: 0 0 0 4px var(--rv-accent-muted), 0 0 0 4px var(--rv-accent)`
- Focus ring disappears on mouse click (`:focus-visible` semantics — applied via JS class `keyboard-nav-active` on `<body>`, set on `keydown`, cleared on `mousedown`)

**Implementation note for executor:** Do not rely on CSS-only `:focus-visible` — react-d3-tree's `foreignObject` may not propagate focus-visible state correctly. Use the `keyboard-nav-active` body class approach.

### InlineRenameInput

Source: 03-CONTEXT.md D-13; EDIT-01.

**Trigger:** Double-click on node title in canvas, or F2 key when node is focused/selected.

**Visual spec:**
- `position: fixed` (not absolute) — placed via `getBoundingClientRect()` of the node's title element + inverse D3 zoom transform
- Width: matches node card width (220px max, same as `RoadmapNodeCard` max-width)
- Height: auto; min-height: 20px
- Background: `var(--rv-bg-input)` (#1a1a1c dark)
- Border: `1px solid var(--rv-border-focus)` (`var(--rv-accent)` value, #4a9eff)
- Border-radius: 4px
- Padding: 4px 4px (tight — overlays the node title area; aligned to 4px grid)
- Font: 13px / 600 / `var(--rv-text-primary)` — exact match to node title
- Box-shadow: `0 0 0 3px var(--rv-accent-muted)` — glow effect to communicate edit state
- z-index: 1000 (above tree SVG, below context menu at 9000)

**Behavior:**
- Pre-selects all text on open (full title selected)
- Enter: commits rename, closes input
- Escape: cancels rename, closes input, restores original title
- Click outside: commits rename
- Input must remain positioned correctly when user pans/zooms canvas — the `fixed` position is recalculated on every canvas transform update while the input is open

| Ctrl+V | Canvas focused | Paste node from clipboard |

| Ctrl+↓ | Canvas | Move node down within siblings |
