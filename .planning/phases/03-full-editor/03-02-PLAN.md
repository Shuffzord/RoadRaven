<design_note>
**Context menu decision: Radix.**

D-02 from CONTEXT.md says: *"Custom webview-rendered `<div>` context menu on ALL platforms — skip Electrobun's native `ContextMenu.showContextMenu()` entirely."*

The researcher's reading (RESEARCH.md §1, §Don't Hand-Roll): D-02's phrase "custom webview-rendered `<div>`" contrasts with the native OS menu (`Electrobun.ContextMenu.showContextMenu()`). Radix's `@radix-ui/react-context-menu` renders a `<div>` inside a React portal to `document.body` — which IS a "custom webview-rendered `<div>`". It is NOT a native OS menu. The Radix menu appears identically on all three platforms (including Linux where Electrobun's native menu is a no-op), thereby also satisfying EDIT-18 with no extra work.

Hand-rolling W3C-correct ARIA, arrow-key cycling, Home/End navigation, typeahead search, roving tabindex, submenu positioning, collision detection, and focus trap is ~200+ LOC with a meaningful bug surface. Radix has shipped this primitive for 4+ years and is already installed (v2.2.16 in package.json).

**Why this satisfies D-02's spirit:**
1. One codepath across all platforms (D-02's explicit goal: "One implementation, consistent behavior, easier to test")
2. Rendered as a `<div>` via React portal (matches D-02's wording literally)
3. Keyboard-navigable and ARIA-compliant (D-02's requirement + EDIT-09)
4. No native OS menu dependency (D-02's reason for the decision — Linux no-op)

The Paste action supports both clipboard-available and in-memory-buffer states; Paste is disabled only when BOTH are unavailable.

**50ms budget:** Radix mounts via portal on trigger; typical 10-30ms render in simple menus. If UAT measurement shows >50ms on the 300-node tree, fallback is `forceMount` + CSS `visibility: hidden` / `visibility: visible` toggled via `data-state`. Gate is checkpoint-verified manually (VALIDATION.md Manual-Only Verifications).
</design_note>

    ```tsx
    import * as ContextMenu from "@radix-ui/react-context-menu";
    import type { ReactNode } from "react";
    import { useRoadmapStore } from "../store/roadmapStore";

    const DEFAULT_STATUSES = [
      { id: "not-started", label: "Not Started" },
      { id: "in-progress", label: "In Progress" },
      { id: "completed", label: "Completed" },
      { id: "blocked", label: "Blocked" },
    ] as const;

    const MENU_SURFACE_CLASS =
      "min-w-[200px] max-w-[280px] py-1 bg-[var(--rv-bg-elevated)] border border-[var(--rv-border)] rounded-[8px] shadow-[var(--rv-shadow-config)] z-[9000] will-change-transform";
    const ITEM_CLASS =
      "flex items-center justify-between h-[28px] px-3 text-[13px] text-[var(--rv-text-primary)] select-none cursor-default outline-none data-[highlighted]:bg-[var(--rv-accent-muted)] data-[highlighted]:text-[var(--rv-text-primary)] data-[disabled]:text-[var(--rv-text-tertiary)] data-[disabled]:pointer-events-none";
    const ITEM_DESTRUCTIVE_CLASS =
      "flex items-center justify-between h-[28px] px-3 text-[13px] text-[var(--rv-status-blocked)] select-none cursor-default outline-none data-[highlighted]:bg-[var(--rv-accent-muted)]";
    const HINT_CLASS = "text-[11px] text-[var(--rv-text-tertiary)] ml-4";
    const SEP_CLASS = "h-px my-1 bg-[var(--rv-border-subtle)]";

    interface RoadRavenContextMenuProps {
      children: ReactNode;
      /** Called on right-click to set target node id (from data-source-id). Null = canvas background. */
      onOpen: (targetNodeId: string | null) => void;
      targetNodeId: string | null;
    }

    export function RoadRavenContextMenu({ children, onOpen, targetNodeId }: RoadRavenContextMenuProps) {
      return (
        <ContextMenu.Root onOpenChange={(open) => { if (!open) onOpen(null); }}>
          <ContextMenu.Trigger
            asChild
            onContextMenu={(e) => {
              const el = (e.target as Element)?.closest?.("[data-source-id]") as HTMLElement | null;
              onOpen(el?.dataset.sourceId ?? null);
            }}
          >
            {children}
          </ContextMenu.Trigger>
          <ContextMenu.Portal>
            <ContextMenu.Content className={MENU_SURFACE_CLASS} aria-label={targetNodeId ? "Node actions" : "Canvas actions"} collisionPadding={8}>
              {targetNodeId ? <NodeMenuItems nodeId={targetNodeId} /> : <CanvasMenuItems />}
            </ContextMenu.Content>
          </ContextMenu.Portal>
        </ContextMenu.Root>
      );
    }

    function NodeMenuItems({ nodeId }: { nodeId: string }) {
      const {
        addChild, addSiblingAbove, addSiblingBelow,
        duplicateNode, copySubtreeToClipboard, pasteFromClipboard,
        moveNodeUp, moveNodeDown, requestDelete,
        lastCopiedSubtree, schema,
      } = useRoadmapStore();
      const statuses = schema?.statusConfig?.length ? schema.statusConfig : DEFAULT_STATUSES;
      const canPaste = lastCopiedSubtree !== null || typeof navigator !== "undefined";

      return (
        <>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => openRename(nodeId)}>
            <span>Rename</span><span className={HINT_CLASS}>F2</span>
          </ContextMenu.Item>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => addChild(nodeId)}>
            <span>Add Child</span><span className={HINT_CLASS}>Enter</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => addSiblingAbove(nodeId)}>
            <span>Add Sibling Above</span><span className={HINT_CLASS}>Shift+Enter</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => addSiblingBelow(nodeId)}>
            <span>Add Sibling Below</span><span className={HINT_CLASS}>Tab</span>
          </ContextMenu.Item>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => duplicateNode(nodeId)}>
            <span>Duplicate</span><span className={HINT_CLASS}>Ctrl+D</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => void copySubtreeToClipboard(nodeId)}>
            <span>Copy</span><span className={HINT_CLASS}>Ctrl+C</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} disabled={!canPaste} onSelect={() => void pasteFromClipboard(nodeId)}>
            <span>Paste</span><span className={HINT_CLASS}>Ctrl+V</span>
          </ContextMenu.Item>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => moveNodeUp(nodeId)}>
            <span>Move Up</span><span className={HINT_CLASS}>Ctrl+↑</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => moveNodeDown(nodeId)}>
            <span>Move Down</span><span className={HINT_CLASS}>Ctrl+↓</span>
          </ContextMenu.Item>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Sub>
            <ContextMenu.SubTrigger className={ITEM_CLASS}>
              <span>Change Status</span>
              <svg width={12} height={12} viewBox="0 0 24 24" fill="none" stroke="currentColor" strokeWidth={2} aria-hidden="true"><polyline points="9 18 15 12 9 6" /></svg>
            </ContextMenu.SubTrigger>
            <ContextMenu.Portal>
              <ContextMenu.SubContent className={MENU_SURFACE_CLASS} aria-label="Change status" collisionPadding={8}>
                {statuses.map(s => (
                  <ContextMenu.Item key={s.id} className={ITEM_CLASS} onSelect={() => useRoadmapStore.getState().updateNodeStatus(nodeId, s.id)}>
                    <span>{s.label}</span>
                  </ContextMenu.Item>
                ))}
              </ContextMenu.SubContent>
            </ContextMenu.Portal>
          </ContextMenu.Sub>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Item className={ITEM_DESTRUCTIVE_CLASS} onSelect={() => requestDelete(nodeId)}>
            <span>Delete</span><span className={HINT_CLASS} style={{ color: "var(--rv-status-blocked)", opacity: 0.6 }}>Del</span>
          </ContextMenu.Item>
        </>
      );
    }

    function CanvasMenuItems() {
      const { pasteFromClipboard, setLayout, layoutOrientation, resetView, lastCopiedSubtree, schema } = useRoadmapStore();
      const canPaste = lastCopiedSubtree !== null;
      const rootId = schema?.nodes?.[0]?.id ?? null;

      return (
        <>
          <ContextMenu.Item className={ITEM_CLASS} disabled={!canPaste} onSelect={() => void pasteFromClipboard(null)}>
            <span>Paste</span><span className={HINT_CLASS}>Ctrl+V</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} disabled={!rootId} onSelect={() => rootId && useRoadmapStore.getState().addChild(rootId)}>
            <span>Add Root Child</span><span className={HINT_CLASS}>Enter</span>
          </ContextMenu.Item>
          <ContextMenu.Separator className={SEP_CLASS} />
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => resetView()}>
            <span>Fit to View</span>
          </ContextMenu.Item>
          <ContextMenu.Item className={ITEM_CLASS} onSelect={() => setLayout(layoutOrientation === "TB" ? "LR" : "TB")}>
            <span>Toggle Layout</span>
          </ContextMenu.Item>
        </>
      );
    }

    // openRename is exported from Canvas via a side-channel — set a global "pending rename" flag on store
    function openRename(nodeId: string) {
      // Trigger via a store field the Canvas subscribes to — set a one-shot request
      useRoadmapStore.getState().setFocusedNode(nodeId);
      // Use a CustomEvent that Canvas listens for — avoids coupling menu to Canvas's useInlineRename
      window.dispatchEvent(new CustomEvent("roadraven:open-rename", { detail: { nodeId } }));
    }
    ```

    Note on `openRename`: the Rename menu item triggers inline rename via a CustomEvent. Canvas (Task 2) subscribes via `window.addEventListener("roadraven:open-rename", handler)` and calls `inlineRename.open(...)` with the node's position. This avoids passing Canvas-local state (transform, container rect) down through the menu.

    Canvas alternative: expose a singleton via a small Zustand-adjacent helper, or pass an `onRenameRequested` prop from Canvas. CustomEvent is simpler and keeps ContextMenu presentational.

    Tests use `userEvent.pointer({ keys: "[MouseRight>]", target: screen.getByTestId("trigger") })` to right-click. Confirm role=menu appears. For the `updateNodeStatus` sub-menu test, right-click → ArrowDown × N → ArrowRight → ArrowDown → Enter and assert `store.getState().nodeIndex.get(testNodeId).status === "completed"`.
