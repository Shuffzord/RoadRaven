## Design North Star

"The Engineered Canvas" — IDE structure with modern polish. Takes structural cues from VS Code and JetBrains (sidebar, panels, status bar) but applies Figma/Linear-inspired refinements: rounded corners, subtle shadows, smooth transitions. The result is a professional tool that doesn't feel cold.

Key attributes: canvas-first layout, configurable density (node edge style, gap, connector), functional typography (monospaced for code/metadata), smooth 150-200ms feedback transitions.

Primary focal point: the canvas area with the rendered node tree — first element the eye lands on; all chrome (top bar, sidebar, status bar, side panel) is visually subordinate.

Source: design.md §1

## Typography

Inter is the sole UI font. Monospace is fallback-only for `<code>` inline blocks in the side panel notes.

Two weights only: **400 (Regular)** for body/notes text; **600 (SemiBold)** for all UI chrome — labels, section headers, buttons, brand, node titles, badges.

| Role | Family | Size | Weight | Line Height | Transform | Usage |
|------|--------|------|--------|-------------|-----------|-------|
| Brand / App name + Panel title | Inter | 14px | 600 (SemiBold) | — | None, letter-spacing -0.01em (brand) | Top bar brand name; side panel header title |
| Node title / Body | Inter | 13px | 400 (Regular) | 1.5 (body) / 1.3 (node title) | None | Node card primary label, side panel notes body text, base page font |
| UI Label / Meta row | Inter | 12px | 600 (SemiBold) | 1.4 | None | Topbar buttons, sidebar file names, panel field values, meta row key+value |
| Section header / Badge / Micro | Inter | 11px | 600 (SemiBold) | 1.2 | Uppercase, tracking 0.05em–0.06em (headers) | Sidebar section headers, panel field labels, status bar text, node status badge pill text, config labels, kbd hints |

Consolidation notes:
- Former 10px (Micro/Meta) → 11px. Former 15px (panel title) → 14px.
- Former weight 500 (Medium) collapsed to 600 throughout UI chrome.
- Weight 400 is used only for body copy and side panel notes text.

Source: variant-c-merged.html CSS font-size/font-weight declarations, design.md §3

### Accent Reserved For

`--rv-accent` (`#4a9eff` dark/light; `#60b0ff` high-contrast) is reserved for:

1. Selected node border ring — `border-color: var(--rv-accent)` + `box-shadow: 0 0 0 1px var(--rv-accent)` on `.node.selected`
2. Brand icon in top bar — SVG `color: var(--rv-accent)` on `.topbar__brand svg`
3. Active layout toggle option — `color: var(--rv-accent)` + `background: var(--rv-accent-muted)` on `.topbar__layout-opt.active`
4. Active theme switcher button — `color: var(--rv-accent)` + `background: var(--rv-accent-muted)` on `.topbar__theme-btn.active`
5. Active config panel toggle option — `color: var(--rv-accent)` on `.config-panel__toggle-opt.active`
6. Focus rings — `outline: 2px solid var(--rv-accent)` on `:focus-visible`
7. Panel title hover — `color: var(--rv-accent)` on `.panel__title:hover`
8. Status badge for In-Progress nodes — `--rv-status-in-progress` shares the same value as accent
9. Copy button hover — `color: var(--rv-accent)` on `.panel__copy-btn:hover`
10. Status select hover border — `border-color: var(--rv-accent)` on `.panel__status-select:hover`
11. Panel type badge — `background: var(--rv-accent-muted)`, `color: var(--rv-accent)` on `.panel__type-badge`
12. Config gap button active — `border-color: var(--rv-accent-border)` on `.config-panel__gap-btn.active`

Accent is NOT used for: body text, secondary labels, inactive borders, general backgrounds, node default borders.

## Interaction Contracts

### Theme Switching

- Trigger: click on one of the three theme buttons in the top bar theme-switcher group; or menu item (Dark / Light / High Contrast / System)
- Mechanism: swap `data-theme` attribute on `<html>` element
- Transition: instant — no CSS transition on color token changes; structural transitions (sidebar collapse, panel slide) use 150-200ms
- System mode: attach `window.matchMedia('(prefers-color-scheme: dark)')` listener; resolve to `dark` or `light` theme value; update reactively when OS setting changes
- Persistence: write `{ theme: "dark" | "light" | "high-contrast" | "system" }` to `.roadmap-settings.json` via Bun RPC `logMessage` / settings channel
- Source: CONTEXT.md D-04, D-05; variant-c-merged.html JS

### Sidebar Collapse

- Trigger: click collapse button in sidebar header (Ctrl+B keyboard shortcut)
- Mechanism: toggle `.collapsed` class on `.sidebar` element; CSS transition `width 200ms ease`
- Collapse icon: rotates 180deg (`transform 200ms ease`)
- State: hidden text labels, centered icons; sections and headers hidden
- Source: variant-c-merged.html sidebar JS

### Side Panel Open/Close

- Open: node click → set `.panel.open` class → CSS transition `width 200ms ease-out` from 0 to 340px
- Close: click close button → remove `.panel.open` class → reverse transition
- Source: variant-c-merged.html panel JS, design.md §4

### Per-Schema Theme Override

- Trigger: schema loaded with `themeConfig` block
- Scope: applied as inline CSS custom properties on a container element wrapping the schema canvas view — NOT on `:root`
- Effect: overrides `--rv-status-*` tokens and `--node-radius` for that file's view only
- Override removal: when schema is unloaded, container's inline styles cleared
- Base theme compatibility: overrides stack on top of active base theme (D-06)
- Source: CONTEXT.md D-07, D-08

### Button / Interactive Element States

- Default: per component spec above
- Hover: background shift, color shift — 150ms ease
- Active/pressed: `--rv-bg-active` background
- Focus visible: `outline: 2px solid var(--rv-accent)`, 2px offset
- Disabled: `--rv-text-tertiary` text, `opacity: 0.5`, no hover response
- Transition duration: 150ms for color/background; 200ms for structural (width, transform)

### Config Panel Toggle

- Trigger: click config toggle button (bottom-right canvas overlay)
- Mechanism: toggle `.config-panel.open` class (`display: none` → `display: block`)
- Transitions: node `border-radius 200ms ease` when node corners configuration changes; node gap changes are instant (layout recalculation)
- Source: variant-c-merged.html config JS

## Accessibility Contract

- All theme buttons: `role="radio"`, `aria-checked`, `aria-label` per button; group: `role="radiogroup"`, `aria-label="Theme switcher"`
- Layout toggle buttons: `role="radio"`, `aria-checked`; group: `role="radiogroup"`, `aria-label="Tree layout direction"`
- Top bar toolbar: `role="toolbar"`, `aria-label="Main toolbar"`
- Sidebar nav: `role="navigation"`, `aria-label="Sidebar navigation"`
- Search: `role="search"` on container, `aria-label="Search nodes"` on input
- Icon-only buttons: `aria-label` required on every instance (collapse sidebar, close panel, zoom, fit view, settings gear)
- Status color: never the sole status indicator — node badge always includes text label + colored dot (PACK-06)
- Focus indicators: `outline: 2px solid var(--rv-accent)`, 2px offset, visible in all three themes including high-contrast
- `prefers-reduced-motion`: Phase 1 structural transitions (sidebar collapse 200ms, panel slide 200ms) should respect `prefers-reduced-motion: reduce` — set durations to 0ms via media query
- Keyboard: theme group and layout toggle groups navigable with arrow keys + Enter (Phase 1 scope — menu items only)
