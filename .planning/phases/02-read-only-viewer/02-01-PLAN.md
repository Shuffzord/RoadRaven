2. Define `toTreeDatum(node: RoadmapNode): RawNodeDatum` conversion function:
   - `name`: `node.title`
   - `attributes`: `{ id: node.id, status: node.status, type: node.type ?? "", notes: node.notes ?? "", createdAt: node.createdAt ?? "", updatedAt: node.updatedAt ?? "" }`
   - `children`: `node.children?.map(toTreeDatum)`

3. Define `buildNodeIndex(nodes: RoadmapNode[]): Map<string, RoadmapNode>` that flattens the tree into a Map keyed by node.id. Recursively walks all children.

4. Define the store interface:
```typescript
interface RoadmapState {
  // Document data
  schema: RoadmapSchema | null;
  filePath: string | null;
  treeData: RawNodeDatum | null;
  dataKey: string;  // string counter — increments only on structural changes
  nodeIndex: Map<string, RoadmapNode>;  // flat id -> node for in-place status updates

  // UI state
  selectedNodeId: string | null;
  layoutOrientation: "TB" | "LR";
  isPanelPinned: boolean;

  // Actions — structural (increment dataKey)
  loadSchema: (schema: RoadmapSchema, filePath: string) => void;
  reloadSchema: (schema: RoadmapSchema) => void;

  // Actions — in-place (no dataKey change)
  updateNodeStatus: (nodeId: string, status: string) => void;
  setSelectedNode: (id: string | null) => void;
  setLayout: (orientation: "TB" | "LR") => void;
  getSelectedNode: () => RoadmapNode | undefined;
  getNodeCount: () => number;
}
```

5. Implement with `create<RoadmapState>((set, get) => ({...}))`:
   - `loadSchema(schema, filePath)`: Build `treeData` from `schema.nodes[0]` via `toTreeDatum()`. Build `nodeIndex` via `buildNodeIndex(schema.nodes)`. Increment `dataKey` (parse current as number, increment, convert back to string). Set `schema`, `filePath`, `treeData`, `nodeIndex`, `dataKey`.
   - `reloadSchema(schema)`: Same as loadSchema but preserves existing `filePath`. Used by file watcher reload.
   - `updateNodeStatus(nodeId, status)`: Get node from `nodeIndex`. If not found, return (no-op). Mutate `node.status = status` in-place. Call `set({})` to trigger subscriber notifications but do NOT create a new treeData reference or increment dataKey. This is the critical performance path per D-02.
   - `setSelectedNode(id)`: `set({ selectedNodeId: id })`
   - `setLayout(orientation)`: `set({ layoutOrientation: orientation })`
   - `getSelectedNode()`: `return get().nodeIndex.get(get().selectedNodeId ?? "")`
   - `getNodeCount()`: `return get().nodeIndex.size`

6. Export: `export { useRoadmapStore, toTreeDatum, buildNodeIndex }` (export helpers for testing).
