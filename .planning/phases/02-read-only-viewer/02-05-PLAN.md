**Gap 2 — Fit View expands collapsed nodes (UAT test 7, severity: minor):**

The root cause is `key={`tree-${viewResetKey}`}` on the Tree component (Canvas.tsx line 144). Changing the React key forces a full unmount/remount of Tree, which destroys react-d3-tree's internal `__rd3t.collapsed` state. On remount, `initialDepth=3` re-expands all nodes.

Fix in `packages/desktop/src/mainview/components/Canvas.tsx`:
1. Remove `viewResetKey` from the `useShallow` selector (lines 22 and 31).
2. Remove the `key={`tree-${viewResetKey}`}` prop from the `<Tree>` component (line 144). The Tree already receives `translate` and `zoomLevel` as props — updating these via `resetView()` will smoothly reposition the viewport without remounting.

Fix in `packages/desktop/src/mainview/store/roadmapStore.ts`:
1. Remove `viewResetKey: number` from the `RoadmapState` interface (line 63).
2. Remove `viewResetKey: 0` from `INITIAL_STATE` (line 104).
3. In `resetView()` (line 169-181), remove the line `viewResetKey: get().viewResetKey + 1` from the `set()` call. Keep the `translate` and `zoomLevel` calculations — those are the actual viewport reset mechanism.

Do NOT add console.logs.
