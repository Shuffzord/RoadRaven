> This project does NOT use shadcn. It uses a custom --rv-* CSS custom property token system with Tailwind v4.
> All values are sourced from existing code and upstream artifacts — no user questions were required.

## Color

All values are --rv-* CSS custom property tokens. Never use hardcoded hex values in component code (THEME-05 requirement enforced in CI).

### 60/30/10 Split (Dark Theme — default)

| Role | Token | Dark Value | Light Value | Usage |
|------|-------|------------|-------------|-------|
| Dominant (60%) | --rv-bg-base / --rv-bg-canvas | #131313 | #ffffff / #fafafa | App background, canvas |
| Secondary (30%) | --rv-bg-surface / --rv-bg-panel / --rv-bg-toolbar | #1b1b1c | #f5f5f5 | Sidebar, side panel, top bar, status bar |
| Accent (10%) | --rv-accent | #4a9eff | #4a9eff | Reserved elements only (see list below) |
| Destructive | --rv-status-blocked | #ef4444 | #ef4444 | Blocked status badge + stripe only |

### Accent Reserved For

`--rv-accent` (#4a9eff) is used **only** on these specific elements:

1. Selected node border ring (1px accent ring on click-selected node)
2. Primary CTA buttons — "Open File" and "New Roadmap" on welcome screen
3. Active layout toggle state (TB/LR toggle group — active segment)
4. Active theme switcher segment (bg-rv-accent-muted + text-rv-accent)
5. Focus ring on all interactive elements (2px outline, via :focus-visible global rule)
6. Type badge in side panel (bg-rv-accent-muted background, text-rv-accent label — source: SidePanel.tsx)
7. Inline code text color in rendered markdown notes
8. Hyperlinks in rendered markdown notes
9. Schema validation error panel — error location highlight (text only, not background)
10. File watcher reload indicator (brief flash on tree container border, 300ms, accent color)

`--rv-accent` is NOT used for: general hover states, connector lines, dot-grid, or decorative elements.

### Status Colors (all themes)

| Status | Stripe / Badge Color | Background | Token |
|--------|---------------------|------------|-------|
| not-started | --rv-status-not-started (#6b6b6b) | --rv-status-not-started-bg | Left stripe + badge dot + badge text |
| in-progress | --rv-status-in-progress (#4a9eff) | --rv-status-in-progress-bg | Left stripe + badge dot + badge text |
| completed | --rv-status-completed (#4ade80) | --rv-status-completed-bg | Left stripe + badge dot + badge text |
| blocked | --rv-status-blocked (#ef4444) | --rv-status-blocked-bg | Left stripe + badge dot + badge text |

Status color must never be the sole indicator — badge pill includes both a colored dot AND a text label (PACK-06 / accessibility).

## Interaction Contracts

### Tree Canvas

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Zoom in | Scroll wheel up | Zoom centered on cursor; built into react-d3-tree |
| Zoom out | Scroll wheel down | Zoom centered on cursor; built into react-d3-tree |
| Pinch zoom | Two-finger trackpad | Standard pinch; built into react-d3-tree |
| Pan | Click + drag on empty canvas | Pans the tree; built into react-d3-tree |
| Fit to view | Click "Fit View" button in TopBar | Calls react-d3-tree translate/zoom reset to fit all nodes |
| Select node | Single click on node | Opens side panel; adds 1px accent border ring to node |
| Collapse subtree | Click collapse button on node | Collapses children; button is small (14px) chevron on node |
| Expand subtree | Click expand button on collapsed node | Expands children |
| Layout toggle TB | Click "TB" in ToggleGroup | Switches orientation; persists to .roadmap-settings.json |
| Layout toggle LR | Click "LR" in ToggleGroup | Switches orientation; persists to .roadmap-settings.json |
| File reload | External file change detected | Tree re-renders; brief accent flash on canvas border (300ms) |
| Deselect node | Click empty canvas | Closes side panel (if not pinned); removes accent ring |

Default collapse depth: nodes beyond depth 3 collapse on initial load (VIEW-04).

### Side Panel

| Interaction | Trigger | Behavior |
|-------------|---------|----------|
| Open | Node click | Panel slides in from right; 200ms ease-out; width 340px |
| Close | Click X button | Panel slides out; 200ms ease-out; width → 0 |
| Pin mode | Viewport width > 1400px | Panel stays open even on canvas click; X button still works |
| Resize | Drag left edge of panel | Width clamps between 320px and 50% viewport; no snap |
| Copy ID | Click copy icon next to node ID | Copies ID to clipboard; icon briefly changes to checkmark (1s) |

Side panel is read-only in Phase 2. No editable fields. Notes section renders markdown via MarkdownRenderer.

### Welcome Screen

| Element | Interaction | Behavior |
|---------|-------------|----------|
| "Open File" button | Click | Opens native file picker via Electrobun; accent-colored primary button |
| "New Roadmap" button | Click | Creates in-memory skeleton (Phase 3 implements save); accent-outlined secondary button |
| Recent file row | Click | Opens that file directly |
| Sample schema link | Click | Opens the bundled schema (hello-world.json or getting-started.json) |

Welcome screen is shown when roadmapStore has no loaded file. It replaces the canvas (not an overlay).

### Schema Error Panel

| State | Trigger | Display |
|-------|---------|---------|
| Validation error | Zod parse fails on file open | Panel overlays bottom of canvas; shows error type, severity, JSON path |
| Multiple errors | Multiple Zod errors | Scrollable list; each error on its own row |
| Dismiss | File reloads successfully | Panel disappears automatically |

Error panel does not block canvas display. If partial data is available, tree renders what it can.

## DataKey Pattern Contract

Critical for 30 fps performance gate (VIEW-11).

| Operation | dataKey behavior | Store update method |
|-----------|-----------------|---------------------|
| File open (new schema) | increment dataKey | Full store replace |
| Structural change (add/delete/move node) | increment dataKey | Full store replace |
| Status-only update (updateNode status field) | DO NOT increment dataKey | In-place zustand set via useShallow selector |
| File watcher reload | increment dataKey | Full store replace |
| $ref file change | increment dataKey | Full store replace |

react-d3-tree deep-clones on every data ref change. Status-only updates must bypass this by keeping the same data reference and updating only the status field in the flat node index.

Source: D-02 (CONTEXT.md), STATE.md decisions, ROADMAP.md Phase 2 Plan 1 description
