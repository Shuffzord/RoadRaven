# Phase 4 — UI Design Contract

> Visual and interaction contract for the Event API phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope: the six UI surfaces this phase touches — (1) status-bar event-api pill, (2) welcome-screen URL line, (3) live pulse on `RoadmapNodeCard`, (4) SidePanel Integration zone, (5) bottom event-log drawer, (6) error toasts. Nothing outside these surfaces is re-specified.
>
> **Source of truth:** `04-CONTEXT.md` D-01 through D-29 are locked. This document converts those decisions into measurable tokens the executor implements against. Where CONTEXT.md gave "Claude's Discretion," this document picks a concrete value.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual, pre-existing) |
| Preset | not applicable — shadcn declined; project uses Tailwind v4 + `--rv-*` custom properties established in Phase 1 |
| Component library | Radix UI primitives (`@radix-ui/react-context-menu`, `@radix-ui/react-dialog`, `@radix-ui/react-dropdown-menu` — already installed) |
| Icon library | none — inline SVG pattern matching existing components (ExternalEditToast, StatusBar, SaveIndicator). Use 12px stroke=2 for inline status glyphs, 14px stroke=2 for toasts, 16px stroke=2 for drawer toolbar icons. |
| Font | Inter, system-ui, -apple-system, sans-serif — loaded globally in `index.css`. No new font additions this phase. |

**Why no shadcn:** project predates shadcn gate; Tailwind v4 `@theme` + `--rv-*` token architecture from Phase 1 is already the design system. Adding shadcn now would fragment the contract (THEME-05 forbids hardcoded colours — shadcn presets ship hardcoded HSL values that would collide with the `--rv-*` token invariant). Registry safety gate: not applicable.

## Spacing Scale

Declared values (all multiples of 4, aligned with existing Phase 1–3 components):

| Token | Value | Usage in Phase 4 |
|-------|-------|------------------|
| xs | 4px | Icon-to-label gap inside status-bar pill, toast icon-to-text gap |
| sm | 8px | Pill internal horizontal padding, row gaps in Integration zone header, drawer row vertical padding |
| md | 12px | Drawer row horizontal padding, toast padding-y, Integration zone section internal padding |
| md+ | 16px | Toast padding-x, Integration zone section margin, drawer toolbar padding |
| lg | 24px | Drawer collapsed header-strip height is 24px (matches context — single "peek" row) |
| lg+ | 32px | Status-bar height stays at 32px (existing, not changed) |
| touch | 28px | Minimum interactive hit height for pill, drawer filter buttons, toast dismiss — provides ≥28px target while fitting the 32px status bar with 2px vertical breathing room |

**Exceptions:**
- **Pulse ring radius:** matches the node card's own `border-radius: 8px` + 4px outer offset — not a spacing token, a geometric continuation of the existing focus-ring pattern.
- **Drawer default height:** 30% of viewport (per D-18). Resize is clamped between 24px (collapsed) and 70% of viewport.
- **Event log row min-height:** 32px (must equal status-bar height for visual rhythm; gives room for 11px text + 4px top/bottom padding + 12px meta badge).

## Typography

Four roles declared, two weights maximum. Inherits body baseline from `index.css` (`font-size: 13px`, `line-height: 1.5`).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| micro | 11px | 400 (regular) | 1.4 | Status-bar pill label (`:47921`), drawer row timestamp/source/meta-preview, welcome-screen URL line, toast secondary text |
| body | 13px | 400 (regular) | 1.5 | Integration zone labels, drawer row nodeId + full meta keys, toast primary message — matches existing SidePanel body |
| label | 13px | 600 (semibold) | 1.4 | Integration zone `● Live` header, drawer toolbar section titles ("Events", filter-bar labels), toast CTAs ("Dismiss") |
| numeric | 11px | 500 (medium) | 1.0 | Producer count badge in status-bar pill (`· 2`), drawer row count header (`[23 events]`) — tabular-nums variant to avoid layout shift when count changes |

**Weights:** exactly 2 active — `400 regular` and `600 semibold`. `500 medium` is used only on the numeric count (tabular-nums) and is considered a within-variant (fallback to 600 acceptable if 500 unavailable in the Inter fallback stack).

**Why not size 16+:** this phase's surfaces are all status-bar / chrome / overlay — none are primary reading surfaces. The largest new text (drawer toolbar title, Integration zone `● Live`) sits at 13px semibold, matching SidePanel heading rhythm. Adding a 16/20px role would break the existing shell typography hierarchy.

| Role | Token | Usage |
|------|-------|-------|

| Secondary (~30%) | `--rv-bg-surface`, `--rv-bg-elevated`, `--rv-bg-hover` | Drawer row backgrounds (zebra via `--rv-bg-surface` on odd rows), Integration zone card interior, toast container, pill idle fill |
| Accent (~10%) | `--rv-accent`, `--rv-accent-muted`, `--rv-accent-border` | **Reserved list below — never "all interactive"** |
| Success (status-completed) | `--rv-status-completed`, `--rv-status-completed-bg` | Live pulse ring colour, `● Live` header dot in Integration zone, status-bar pill dot when server is listening |
| Destructive | `--rv-status-blocked`, `--rv-status-blocked-bg` | Error row left stripe (4px) in drawer, error-toast left stripe, error badge in Integration zone header when last event has `_error` |
| Neutral-idle | `--rv-text-tertiary`, `--rv-border-subtle` | `○ Event API off` pill, `— No events received` empty-state headline, disabled filter chips |

### Accent reserved-for list (exhaustive, no additions without contract revision)

1. **Status-bar pill, server-listening state:** the `●` dot uses `--rv-status-completed` (green), NOT accent. Accent (`--rv-accent`) is applied only when the user hovers the pill to signal "click-to-interact" — hover background `--rv-accent-muted`, border `--rv-accent-border`.
2. **Drawer toggle button in TopBar:** button-idle is `--rv-text-secondary`. When the drawer is open (active state), the button's icon + label text switch to `--rv-accent` and the button gets a 1px bottom underline in `--rv-accent` — same pattern as active tabs elsewhere in the shell.
3. **Integration zone "Open full log" link:** text `--rv-accent`; hover → `--rv-accent-hover`; no underline until hover.
4. **Event log row when `selectedNodeId === row.nodeId`:** left-edge 2px accent rule (`--rv-accent`) + row background `--rv-accent-muted`. Differs from error rows (error uses 4px `--rv-status-blocked` stripe, accent uses 2px to avoid visual collision when a row is both selected AND errored — error stripe takes the 4px lane, accent rule sits inside).
5. **Focus-visible outline on every new interactive element** (pill, drawer toggle, drawer row, filter chips, toast dismiss button) — inherits global `:focus-visible` rule (`outline: 2px solid var(--rv-accent); outline-offset: 2px;`). Not counted against the 10% budget because it's state-driven, not decorative.

Accent is **NOT** used for: filter-bar active chip background (use `--rv-bg-active`), drawer scrollbar thumb (use `--rv-scrollbar-thumb`), pulse animation target colour (uses `--rv-status-completed` — green = "alive", consistent with VIEW-06 status semantics).

|-------|-----------|--------------|-----------------|--------|

### Pulse animation — exact specification

Applies to `RoadmapNodeCard` when the store-derived `data-live="true"` attribute is set (D-14, D-15).

```css
@keyframes rv-node-pulse {
  0%   { box-shadow: 0 0 0 0px var(--rv-status-completed-bg), var(--rv-shadow-node); }
  50%  { box-shadow: 0 0 0 6px var(--rv-status-completed-bg), var(--rv-shadow-node); }
  100% { box-shadow: 0 0 0 0px var(--rv-status-completed-bg), var(--rv-shadow-node); }
}

.node[data-live="true"] {
  animation: rv-node-pulse 1600ms ease-in-out infinite;
}

@media (prefers-reduced-motion: reduce) {
  .node[data-live="true"] {
    animation: none;
    /* Static replacement: a 2px solid ring in completed-green, same radius as the card */
    outline: 2px solid var(--rv-status-completed);
    outline-offset: 2px;
  }
}
```

Details:
- **Duration:** 1600ms full cycle — slower than a heartbeat, not distracting when 10+ nodes pulse simultaneously.
- **Timing function:** `ease-in-out` — breathes rather than blinks.
- **Colour:** `--rv-status-completed-bg` (translucent green) expanding to 6px outer ring then collapsing — uses the same green semantic as "completed" status without being mistaken for a completed node (the static status stripe on the left edge still tells the true status; pulse is an orthogonal "fresh event" signal).
- **Reduced-motion fallback:** static 2px solid ring in `--rv-status-completed` — present in `index.css`'s global `prefers-reduced-motion` rule already covers `animation-duration: 0ms !important`, but the static outline is defined here explicitly so the node still reads as "live" under reduced motion.
- **Coexistence with selection/focus rings (D-06 dashed focus + solid selection):** pulse uses `box-shadow`, selection uses `outline` + `box-shadow`, focus uses `outline`. When all three stack: outline (focus/selection) is innermost, then pulse ring at 6px offset. Verified against existing `.keyboard-nav-active .node[data-focused="true"][data-selected="true"]` box-shadow stack — pulse adds one layer, no conflict.

### Error toasts (D-22, D-23, D-24)

Shared container styling: fixed bottom-right (offset 16px from drawer if open, 48px above status bar if drawer closed — matches `ExternalEditToast`'s established `bottom: 48` pattern but bottom-right, not bottom-centre). 1px border `--rv-border`, 4px left stripe `--rv-status-blocked` (error) or `--rv-text-tertiary` (info-style disconnect notice), border-radius 8px, box-shadow `--rv-shadow-config`, padding 12px 16px, gap 12px, zIndex 9000.

| Trigger | Headline | Body | CTA |
|---------|----------|------|-----|
| Malformed event | `Invalid event from {source}.` | `See event log for details.` | `Dismiss` (11px, `--rv-accent`, plain background). No "Retry" per D-22. |
| Unknown nodeId | `Event for unknown node from {source}.` | `Node id not found in the current roadmap.` | `Dismiss` |
| Status not in statusConfig | `Unknown status '{s}' from {source}.` | `Extend statusConfig in the schema to accept this status.` | `Dismiss` |
| Producer disconnected | `Producer {source} disconnected.` | (no body line) | `Dismiss` — info stripe (grey), not error (red) |
| Throttled (≥2 same-type + same-source within 5s, per D-24) | `{N} invalid events from {source}.` | `See event log for details.` | `Dismiss` — count auto-updates in place as additional errors arrive inside the 5s window |

Behaviour:
- **Dismiss-only:** PLUG-06's original "retry" wording is overridden by D-22; no retry button.
- **Throttle merge:** same `{error_type, source}` pair within 5s of a live toast → merge into existing toast, increment count, bump toast dismiss-timer. Separate error types or separate sources → new toast (max 3 toasts stacked vertically; 4th onward pushes oldest out with 200ms fade).
- **Auto-dismiss:** none — toasts are persistent until dismissed. Rationale: errors matter; silent auto-disappear defeats the attention purpose.
- **Keyboard dismiss:** `Escape` dismisses the top-most toast when focus is inside the toast or when no other modal is active.

### Destructive actions

**This phase has zero destructive actions.** The event-log drawer is read-only view of the `.events.jsonl` sidecar; no clear-log button, no delete-event action, no sidecar-wipe. CONTEXT.md `<deferred>` explicitly defers log compaction/rotation, so no confirmation dialogs are required. The dismiss-only toast model (D-22) also means no action invocation exists that requires confirmation.
